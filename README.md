# 클로드 코드(Claude Code)로 팀 자동화 완성 : 에이전트와 원격 제어 실전 가이드

---

## 이 책은 무엇인가요?

이 책은 **Claude AI를 혼자 쓰는 것에서 벗어나, 여러 명의 AI를 팀처럼 동시에 운용하는 방법**을 다룬 실전 가이드입니다.

보통 AI를 이렇게 씁니다.

```
나 → Claude에게 질문 → 답변 → 또 질문 → ...
```

이 책은 이걸 가능하게 합니다.

```
나 (스마트폰으로 지시)
  ↓
팀장 AI → 기획자 AI, 개발자 AI, 리뷰어 AI에게 동시에 배분
```

**핵심 기술 3가지:**

- **Claude Code** — 터미널에서 파일을 직접 읽고 쓰는 AI
- **TMUX** — 화면을 여러 칸으로 나눠 AI를 여러 명 실행
- **Remote-Control** — 스마트폰에서 내 컴퓨터의 AI를 원격 지휘

**한 줄 요약**: *6명짜리 AI 개발팀을 내 컴퓨터에 만들고, 어디서든 스마트폰으로 지휘한다.*

---

## 이런 것들이 가능해집니다

**출근길 지하철에서 일을 시킨다**
스마트폰으로 "결제 모듈 버그 수정해줘"라고 보내면, 사무실에 도착했을 때 AI가 이미 코드를 고쳐놓고 있습니다.

**6가지 일이 동시에 진행된다**
기획자 AI가 설계 문서를 쓰는 동안, 개발자 AI가 코드를 짜고, 리뷰어 AI가 품질을 검토합니다. 순서를 기다릴 필요가 없습니다.

**대화가 끊겨도 작업이 이어진다**
새 세션을 시작해도 AI가 이전 작업 맥락을 파일에서 불러옵니다. "아까 어디까지 했더라"가 사라집니다.

**배포까지 AI가 챙긴다**
코드 작성 → 테스트 → 리뷰 → PR 생성까지 한 흐름으로 이어집니다. 사람은 최종 승인만 합니다.

---

## 어떻게 가능한가요?

세 가지 기술을 조합합니다.

**TMUX — AI를 여러 명으로 만든다**
터미널 화면을 6칸으로 나누고, 각 칸에서 Claude AI를 독립적으로 실행합니다. 각자 다른 역할을 맡아 동시에 일합니다.

![팀 구성 개념](./assets/01-2-team-concept.png)

**CLAUDE.md — AI에게 역할을 부여한다**
각 AI가 시작할 때 읽는 설정 파일입니다. "너는 개발자야, 설계는 네 역할이 아니야"처럼 역할 경계를 명확히 합니다.

**Remote-Control — 스마트폰을 리모컨으로 만든다**
내 컴퓨터에서 실행 중인 AI에게 스마트폰으로 연결합니다. 코드와 파일은 절대 클라우드로 나가지 않고, 통신 경로만 인터넷을 씁니다.

## 책 구성

| 장 | 제목 | 배우는 것 |
|----|------|----------|
| 1장 | 들어가며 | 왜 팀 에이전트가 필요한가, 전체 구조 미리 보기 |
| 2장 | 환경 설치 | Ubuntu/WSL2, Claude Code, TMUX 설치 |
| 3장 | 팀 구성 | TMUX로 6칸 만들기, CLAUDE.md로 역할 부여 |
| 4장 | Remote-Control | 원격 제어 활성화, 서버 모드 설정 |
| 5장 | 모바일 연결 | 스마트폰 앱으로 세션 접속, 푸시 알림 |
| 6장 | 핵심 도구 | gstack, superpowers, gsd, RTK 설치·활용 |
| 7장 | 실전 운용 | 팀 지시 흐름, Bot Mode, Triple Crown 전략 |
| 8장 | 문제 해결 | 인증 오류, 세션 복구, 충돌 방지 |
| 9장 | 마치며 | 앞으로의 발전 방향 |

각 장은 이전 장 위에 쌓이는 구조입니다. 2장에서 기반을 만들고, 3~5장에서 팀을 구성하고, 6~7장에서 실전 수준으로 끌어올립니다.

---

## 이 책을 마치면

**기술적으로 갖추게 되는 것**
- Ubuntu + TMUX + Claude Code로 구성된 6인 AI 팀 환경
- 스마트폰에서 언제든 접속할 수 있는 원격 제어 시스템
- gstack · superpowers · gsd를 조합한 Triple Crown 워크플로우

**일하는 방식이 바뀌는 것**
- "AI에게 물어보는" 사람에서 "AI 팀을 지휘하는" 사람으로
- 자리를 비워도 일이 진행되는 환경
- 혼자서 기획·개발·리뷰를 동시에 돌리는 1인 개발 체계

**이 책이 진짜 가르치는 것**
도구는 계속 바뀝니다. TMUX 대신 다른 것이 나올 수 있고, Claude 다음에 더 좋은 AI가 나올 수 있습니다. 하지만 **역할 분리, 병렬 실행, 원격 제어, 맥락 유지**라는 개념은 어떤 시대에도 유효합니다. 이 책은 그 개념을 가장 현실적인 도구로 직접 구현해보는 과정입니다.
