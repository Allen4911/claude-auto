## 06-4. RTK — Rust Token Killer 토큰 최적화

## RTK란?

**RTK(Rust Token Killer)**는 Claude Code에서 실행되는 CLI 명령어의 출력을 필터링하여 불필요한 토큰 소비를 줄여주는 프록시 도구입니다. Rust로 작성되어 처리 속도가 빠르며, Claude Code의 훅(hook) 시스템과 연동하여 **개발자가 의식하지 않아도 자동으로 동작**합니다.

토큰 절약이 왜 중요한지부터 짚어 봅니다. Claude Code는 실행한 명령어의 출력을 컨텍스트로 읽습니다. `ls` 명령어 하나가 수백 개의 파일 목록을 출력하거나, `git log` 명령어가 수천 줄의 히스토리를 쏟아내면 그 모두가 토큰을 소비합니다. RTK는 이런 출력을 스마트하게 요약하거나 필터링하여 **같은 정보를 더 적은 토큰으로 전달**합니다.

> 💡 **RTK는 투명한 번역기에 가깝습니다.** 외국어 문서에서 핵심만 뽑아 요약하는 번역가처럼, RTK는 장황한 명령어 출력에서 Claude가 실제로 필요한 정보만 추려 전달합니다. Claude는 번역된 결과만 보기 때문에 원문이 얼마나 길었는지 알지 못합니다.

> 💡 **프록시(proxy)란?** 원래 명령어와 Claude 사이에 끼어드는 중간 처리자입니다. 개발자가 `git log`를 요청하면 RTK가 먼저 그 출력을 받아 필요한 부분만 남긴 뒤 Claude에 전달합니다. 개발자 입장에서는 아무것도 바뀐 것처럼 보이지 않는 투명한 중간자입니다.

실제 사용 데이터를 보면 RTK를 적용한 뒤 평균 **63.1%의 토큰**을 아낍니다.

## 설치 방법

### 사전 요구사항

- Rust 툴체인 (rustup) — 소스 빌드 시 필요
- Claude Code 설치 및 실행 중

### Rust 툴체인 설치

#### Windows (WSL2 Ubuntu 터미널) / Linux

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# 설치 확인
rustc --version
cargo --version
```

#### macOS

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# 또는 Homebrew 사용
brew install rust
```

### RTK 설치

#### 방법 1: 사전 빌드된 바이너리 (권장 — Windows/Linux/macOS 공통)

```bash
curl -sSL https://github.com/rtk-ai/rtk/releases/latest/download/install.sh | bash
```

설치 후 바이너리는 `~/.local/bin/rtk` 경로에 위치합니다.

#### 방법 2: 소스에서 빌드 (Windows/Linux/macOS 공통)

```bash
# RTK 저장소 클론
git clone https://github.com/rtk-ai/rtk
cd rtk

# 빌드 및 설치
cargo install --path .
```

> 💡 **소스 빌드가 느린 이유.** Rust는 컴파일 언어라 처음 빌드 시 몇 분이 걸릴 수 있습니다. 한 번 빌드하면 이후 실행은 매우 빠릅니다. 빌드 시간이 부담스러우면 방법 1(바이너리 설치)을 선택하세요.

### 설치 확인

```bash
rtk --version
# 출력: rtk 0.27.2

which rtk
# 출력: /home/user/.local/bin/rtk   (Linux/WSL2)
# 출력: /Users/user/.local/bin/rtk  (macOS)
```

위 두 명령의 출력에 `rtk`와 버전 번호(예: `0.27.2`), 그리고 올바른 설치 경로가 함께 보이면 정상적으로 설치된 것입니다.

> **주의**: `rtk`라는 이름을 가진 다른 패키지(Rust Type Kit)가 존재합니다. `rtk gain` 명령어가 정상 동작하는지 확인하여 올바른 RTK가 설치되었는지 검증하세요.

**설치 직후 첫 동작 확인:**

```bash
# 1. 버전 확인
rtk --version

# 2. 올바른 RTK인지 확인 (이 명령이 "command not found" 없이 실행되면 정상)
rtk gain

# 3. 실제 필터링 효과 확인 (RTK 없을 때 vs 있을 때 비교)
git log --oneline -50 | wc -l       # 원래 출력 줄 수
rtk proxy git log --oneline -50 | wc -l  # RTK 통과 후 줄 수
```

## Hook 기반 자동 적용

RTK에서 가장 쓸모 있는 기능은 **Claude Code 훅과의 통합**입니다. 한 번 설정해 두면 이후 모든 Bash 명령어가 자동으로 RTK를 거쳐 실행됩니다. 개발자가 `git status`를 실행하면 Claude Code 내부에서 `rtk git status`로 투명하게 변환됩니다.

> 💡 **훅(hook)이란?** 특정 동작이 일어날 때 자동으로 끼어들어 실행되는 장치입니다. 여기서는 "Bash 명령 실행 직전(PreToolUse)"에 훅이 끼어들어, 원래 명령을 RTK로 감싸 실행합니다. 한 번 설정하면 개발자가 매번 `rtk`를 붙이지 않아도 자동 적용됩니다.

> 💡 **비유: 자동 세금 공제.** 급여가 통장에 들어오기 전에 세금이 자동 차감되듯, RTK 훅은 명령 출력이 Claude에 전달되기 전에 자동으로 토큰을 절약합니다. 개발자(Claude)는 "이미 처리된" 결과만 받아봅니다.

Claude Code의 `settings.json`에 다음 훅을 추가합니다.

#### settings.json 위치

| 플랫폼 | 경로 |
|--------|------|
| Windows (WSL2) | `~/.claude/settings.json` (Ubuntu 홈 디렉토리) |
| Linux | `~/.claude/settings.json` |
| macOS | `~/.claude/settings.json` |

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/rtk-wrapper.sh"
          }
        ]
      }
    ]
  }
}
```

**훅 설정 단계별 따라하기:**

1. `~/.claude/settings.json`을 열거나 생성합니다.
2. 위 JSON 블록을 붙여넣습니다. 이미 다른 설정이 있다면 `"hooks"` 키만 추가합니다.
3. `~/.claude/hooks/rtk-wrapper.sh` 파일이 존재하는지 확인합니다. (RTK 설치 시 자동 생성됩니다.)
4. Claude Code를 재시작합니다.
5. Claude에게 `git status`를 요청해 보고, RTK가 개입했는지 `rtk gain`으로 확인합니다.

훅이 설정되면 다음과 같이 동작합니다.

```
개발자 요청: "현재 파일 목록 보여줘"
Claude 실행: git status
  ↓ RTK 훅 자동 개입
실제 실행: rtk git status
  ↓ 출력 필터링
Claude 수신: 최적화된 출력 (토큰 65% 절약)
```

위 흐름처럼 개발자나 Claude가 입력한 원래 명령은 훅을 거쳐 RTK로 감싸여 실행되고, 그 출력만 토큰이 절약된 형태로 Claude에 전달됩니다. 개발자는 이 변환 과정을 따로 의식할 필요가 없습니다.

**훅이 작동하는지 확인하는 법:**

```bash
# Claude Code 세션 중 다음을 실행해 보세요
rtk gain
# 이전에 0이었던 "Total commands"가 늘어나 있으면 훅이 정상 작동 중입니다
```

## 주요 메타 명령어

### rtk gain — 토큰 절약 분석

```bash
rtk gain
```

지금까지의 RTK 사용 현황을 보여줍니다.

예를 들어 다음과 같은 요약을 출력합니다.

```
RTK Token Savings (Global Scope)
════════════════════════════════════════

Total commands:    1271
Input tokens:      1.3M
Output tokens:     497.8K
Tokens saved:      850.1K (63.1%)
Total exec time:   29m44s (avg 1.4s)
Efficiency meter: ███████████████░░░░░░ 63.1%

By Command
────────────────────────────────────────────
  #  Command         Count   Saved   Avg%
  1. rtk ls            431  323.6K  65.1%
  2. rtk read          330  137.9K   6.1%
  3. rtk git log ...     1   58.7K  99.9%
```

**출력 해석 방법:**

| 항목 | 의미 |
|------|------|
| Total commands | RTK가 처리한 명령어 총 횟수 |
| Input tokens | RTK 처리 전 원래 출력의 토큰 수 |
| Output tokens | RTK 처리 후 Claude에 전달된 토큰 수 |
| Tokens saved | 절약된 토큰 수와 비율 |
| By Command | 명령어별 절약 현황 (절약량 순 정렬) |

명령어별로 얼마나 절약했는지, 가장 효과가 큰 명령어는 무엇인지 한눈에 파악할 수 있습니다. `read` 명령어의 절약률이 6.1%로 낮은 것은 파일 내용 자체는 요약할 수 없기 때문이고, `git log`가 99.9%로 높은 것은 전체 커밋 히스토리를 핵심 내용만 남기도록 대폭 압축하기 때문입니다.

### rtk gain --history — 히스토리 분석

```bash
rtk gain --history
```

시간 흐름에 따른 토큰 절약 추이를 확인합니다. 특정 기간 동안 어떤 작업에서 토큰을 많이 사용했는지 분석할 수 있습니다.

> 💡 **언제 유용한가?** 프로젝트 기간별로 어느 단계에서 토큰 소비가 많았는지 파악할 때 유용합니다. "지난주에는 git 탐색이 많았구나"라는 패턴을 발견하면 작업 방식을 조정하는 데 참고할 수 있습니다.

### rtk discover — 최적화 기회 발굴

```bash
rtk discover
```

Claude Code 사용 히스토리를 분석하여 RTK를 아직 적용하지 않은 고비용 명령어를 찾아냅니다. "이 명령어도 RTK로 처리하면 X토큰을 추가로 절약할 수 있습니다"라는 형태로 개선 기회를 알려줍니다.

> 💡 **처음 설치했다면 `rtk discover`부터 실행하세요.** 기존 히스토리를 분석해 "어떤 명령어에서 RTK 효과가 클지" 미리 알려주기 때문에, 훅 설정 전에 잠재적 절약 규모를 가늠해볼 수 있습니다.

### rtk proxy — 원시 실행 (디버깅용)

```bash
rtk proxy git log --oneline -20
```

RTK 필터링 없이 원래 명령어 그대로 실행합니다. RTK가 출력을 어떻게 변환하는지 비교하거나 필터링 로직을 디버깅할 때 사용합니다.

**before/after 비교 예시:**

```bash
# RTK 없이 (원본 출력)
rtk proxy git log --oneline -20
# → 20줄 전체 출력

# RTK 적용 후 (필터링된 출력)
rtk git log --oneline -20
# → 핵심 커밋만 압축된 출력 (토큰 대폭 절약)
```

## RTK가 최적화하는 명령어들

RTK는 각 명령어의 특성에 맞게 다른 필터링 전략을 적용합니다.

| 명령어 | 절약률 | 최적화 방식 |
|:---|:---|:---|
| `ls` | ~65% | 불필요한 메타데이터 제거, 핵심 파일명만 유지 |
| `git` | ~99% | 커밋 수 제한, 관련 없는 히스토리 요약 |
| `find` | ~70%+ | 콤팩트 트리 출력, 불필요한 경로 압축 |
| `read` (파일 읽기) | ~6% | 중복 공백/빈 줄 제거 |
| `diff` | ~60%+ | 변경 줄만 추출, 컨텍스트 최소화 |
| `grep` | ~50%+ | 공백 제거, 파일별 그룹핑, 결과 트리밍 |
| `docker` / `kubectl` | ~60%+ | 컴팩트 출력, 불필요한 헤더 제거 |
| `tsc` / `vitest` / `next` | ~80%+ | 실패/에러만 추출, ASCII 아트 제거 |

위 표에서 보듯 `git` 계열 명령이 절약률이 가장 높고(약 99%), 파일 읽기처럼 이미 간결한 출력은 절약 여지가 작습니다(약 6%). 출력이 장황한 명령일수록 RTK의 효과가 큽니다.

절약률이 명령어마다 이렇게 갈리는 데는 이유가 있습니다.

- **`git`이 99%인 이유**: 수천 줄의 커밋 히스토리 중 Claude가 실제로 필요한 것은 최근 수십 줄뿐입니다. 나머지는 모두 제거해도 작업에 영향이 없습니다.
- **`read`가 6%인 이유**: 파일 내용은 그 자체가 정보이므로 함부로 삭제할 수 없습니다. 공백·빈 줄 정도만 정리합니다.
- **`tsc`/`vitest`가 80%인 이유**: 빌드/테스트 출력에는 진행 바, ASCII 아트, 성공한 테스트 목록 등 "결과" 이외의 장식이 많습니다. RTK는 실패한 항목과 에러 메시지만 남깁니다.

## 실전 활용 팁

### 큰 프로젝트에서 특히 효과적

파일 수가 많은 프로젝트에서 `ls`나 `find` 명령어를 자주 실행할 때 RTK의 효과가 극대화됩니다. 수백 개의 파일 목록이 핵심 정보만 담긴 요약으로 변환됩니다.

**예시:** 모노레포에서 `find . -name "*.ts" | wc -l` 결과가 2,000줄이라면, RTK는 이를 구조화된 트리 형태로 요약해 수십 줄로 압축합니다.

### Git 집중 작업 시

`git log`, `git diff`, `git blame` 등 Git 관련 명령어는 RTK를 통해 거의 99%에 가까운 절약이 가능합니다. 특히 전체 히스토리를 훑는 작업에서 효과가 큽니다.

> 💡 **`git log`의 위험성.** 아무 옵션 없이 `git log`를 실행하면 프로젝트 전체 커밋 히스토리가 무제한으로 쏟아져 나옵니다. 오래된 프로젝트라면 수만 줄이 될 수도 있습니다. RTK는 이 출력을 자동으로 수십 줄로 잘라내어 컨텍스트 한계 초과를 방지합니다.

### API 응답 처리 시

외부 API를 `curl`로 호출하고 결과를 Claude에게 전달할 때, RTK가 대용량 JSON 응답을 자동으로 정제합니다.

### 빌드·테스트 자동화 시

`vitest`, `jest`, `tsc`, `next build` 같은 명령어는 성공 시에도 수백 줄을 출력합니다. RTK는 실패한 테스트와 에러만 남기고 나머지는 제거하여, Claude가 문제에 집중할 수 있게 합니다.

> 💡 **"오류만 보여줘"의 자동화.** 테스트 결과에서 "어디서 실패했어?"만 알면 될 때, RTK가 자동으로 그 부분만 추려줍니다. Claude에게 매번 "에러 부분만 봐줘"라고 말하지 않아도 됩니다.

## 토큰 절약의 실제 가치

Claude Code API는 토큰 기반으로 과금됩니다. RTK로 63.1%의 토큰을 절약한다는 것은:

- 같은 비용으로 약 **2.7배** 더 많은 작업을 수행할 수 있음
- 또는 같은 양의 작업을 **37%의 비용**으로 처리할 수 있음

**구체적 계산 예시:**

| 상황 | RTK 없음 | RTK 있음 |
|------|---------|---------|
| 하루 1,000개 명령 실행 | 토큰 100만 개 소비 | 토큰 37만 개 소비 |
| 월 30일 사용 시 | 토큰 3,000만 개 | 토큰 1,110만 개 |
| 절약 토큰 수 | — | 1,890만 개 절약 |

장기 프로젝트, 대용량 코드베이스, 반복적인 파일 탐색이 많은 작업일수록 RTK의 ROI는 높아집니다.

## 요약

RTK는 설치하고 훅을 연결하면 이후 별도의 조작 없이 자동으로 동작합니다. 개발자 입장에서는 투명한 도구이며, 실제로 절약되는 토큰의 양은 `rtk gain` 명령어로 언제든 확인할 수 있습니다. Claude Code를 오래 쓸 계획이라면 초기에 RTK를 설치해 두는 편이 비용을 가장 크게 아끼는 길입니다.

**시작 체크리스트:**
- [ ] RTK 설치 (`rtk --version` 확인)
- [ ] Claude Code 이름충돌 확인 (`rtk gain`이 동작하는지 확인)
- [ ] `settings.json`에 PreToolUse 훅 등록
- [ ] Claude Code 재시작 후 작업 진행
- [ ] 하루 사용 후 `rtk gain`으로 절약량 확인
