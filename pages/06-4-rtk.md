# 6-4. RTK — Rust Token Killer 토큰 최적화

## RTK란?

**RTK(Rust Token Killer)**는 Claude Code에서 실행되는 CLI 명령어의 출력을 필터링하여 불필요한 토큰 소비를 줄여주는 프록시 도구입니다. Rust로 작성되어 처리 속도가 빠르며, Claude Code의 훅(hook) 시스템과 연동하여 **개발자가 의식하지 않아도 자동으로 동작**합니다.

왜 토큰 절약이 중요할까요? Claude Code는 실행한 명령어의 출력을 컨텍스트로 읽습니다. `ls` 명령어 하나가 수백 개의 파일 목록을 출력하거나, `git log` 명령어가 수천 줄의 히스토리를 쏟아내면 그 모두가 토큰을 소비합니다. RTK는 이런 출력을 스마트하게 요약하거나 필터링하여 **같은 정보를 더 적은 토큰으로 전달**합니다.

실제 사용 데이터에 따르면 RTK 적용 후 평균 **63%의 토큰 절약**이 이루어집니다.

---

## 설치 방법

### 사전 요구사항

- Rust 툴체인 (rustup)
- Claude Code 설치 및 실행 중

### 설치 과정

```bash
# RTK 저장소 클론
git clone https://github.com/[rtk-repo]/rtk
cd rtk

# 빌드 및 설치
cargo install --path .

# 또는 사전 빌드된 바이너리 직접 설치
curl -sSL https://[rtk-releases]/install.sh | bash
```

설치 후 바이너리는 `~/.local/bin/rtk` 경로에 위치합니다.

```bash
# 설치 확인
rtk --version
# 출력: rtk 0.27.2

which rtk
# 출력: /home/user/.local/bin/rtk
```

```bash
$ rtk --version

rtk 0.27.2
```

> **주의**: `rtk`라는 이름을 가진 다른 패키지(Rust Type Kit)가 존재합니다. `rtk gain` 명령어가 정상 동작하는지 확인하여 올바른 RTK가 설치되었는지 검증하세요.

## Hook 기반 자동 적용

RTK의 가장 강력한 기능은 **Claude Code 훅과의 통합**입니다. 한 번 설정하면 이후 모든 Bash 명령어가 자동으로 RTK를 통해 실행됩니다. 개발자가 `git status`를 실행하면 Claude Code 내부에서 `rtk git status`로 투명하게 변환됩니다.

Claude Code의 `settings.json`에 다음 훅을 추가합니다.

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.local/bin/rtk-hook"
          }
        ]
      }
    ]
  }
}
```

훅이 설정되면 다음과 같이 동작합니다.

```
개발자 요청: "현재 파일 목록 보여줘"
Claude 실행: git status
  ↓ RTK 훅 자동 개입
실제 실행: rtk git status
  ↓ 출력 필터링
Claude 수신: 최적화된 출력 (토큰 65% 절약)
```

## 주요 메타 명령어

RTK는 몇 가지 직접 실행하는 메타 명령어를 제공합니다.

### rtk gain — 토큰 절약 분석

rtk gain

### rtk gain --history — 히스토리 분석

```bash
rtk gain --history
```

시간 흐름에 따른 토큰 절약 추이를 확인합니다. 특정 기간 동안 어떤 작업에서 토큰을 많이 사용했는지 분석할 수 있습니다.

### rtk discover — 최적화 기회 발굴

```bash
rtk discover
```

Claude Code 사용 히스토리를 분석하여 RTK를 아직 적용하지 않은 고비용 명령어를 찾아냅니다. "이 명령어도 RTK로 처리하면 X토큰을 추가로 절약할 수 있습니다"라는 형태로 개선 기회를 알려줍니다.

### rtk proxy — 원시 실행 (디버깅용)

```bash
rtk proxy git log --oneline -20
```

RTK 필터링 없이 원래 명령어 그대로 실행합니다. RTK가 출력을 어떻게 변환하는지 비교하거나 필터링 로직을 디버깅할 때 사용합니다.

## RTK가 최적화하는 명령어들

RTK는 각 명령어의 특성에 맞게 다른 필터링 전략을 적용합니다.

| 명령어 | 절약율 | 최적화 방식 |
|:---|:---|:---|
| `ls` | ~65% | 불필요한 메타데이터 제거, 핵심 파일명만 유지 |
| `git log` | ~99% | 커밋 수 제한, 관련 없는 히스토리 요약 |
| `curl` | ~99% | 대용량 응답 트리밍, 핵심 데이터만 추출 |
| `read` (파일 읽기) | ~6% | 중복 공백/빈 줄 제거 |
| `cat` | ~10~40% | 파일 크기에 따라 가변 압축 |

## 실전 활용 팁

### 큰 프로젝트에서 특히 효과적

파일 수가 많은 프로젝트에서 `ls`나 `find` 명령어를 자주 실행할 때 RTK의 효과가 극대화됩니다. 수백 개의 파일 목록이 핵심 정보만 담긴 요약으로 변환됩니다.

### Git 집중 작업 시

`git log`, `git diff`, `git blame` 등 Git 관련 명령어는 RTK를 통해 거의 99%에 가까운 절약이 가능합니다. 특히 전체 히스토리를 훑는 작업에서 효과가 큽니다.

### API 응답 처리 시

외부 API를 `curl`로 호출하고 결과를 Claude에게 전달할 때, RTK가 대용량 JSON 응답을 자동으로 정제합니다.

---

## 토큰 절약의 실제 가치

Claude Code API는 토큰 기반으로 과금됩니다. RTK로 63%의 토큰을 절약한다는 것은:

- 같은 비용으로 약 2.7배 더 많은 작업을 수행할 수 있음
- 또는 같은 양의 작업을 37%의 비용으로 처리할 수 있음

장기 프로젝트, 대용량 코드베이스, 반복적인 파일 탐색이 많은 작업일수록 RTK의 ROI는 높아집니다.

---

## 요약

RTK는 설치하고 훅을 연결하면 이후 별도의 조작 없이 자동으로 동작합니다. 개발자 입장에서는 투명한 도구이며, 실제로 절약되는 토큰의 양은 `rtk gain` 명령어로 언제든 확인할 수 있습니다. Claude Code를 장기적으로 사용할 계획이라면 초기에 RTK를 설치하는 것이 비용 대비 가장 효율적인 투자 중 하나입니다.
