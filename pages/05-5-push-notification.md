## 5-5. 푸시 알림으로 작업 완료 확인

## 이 장에서 배울 내용

Claude에게 오래 걸리는 작업을 맡겼을 때, 스마트폰을 계속 들여다볼 필요가 없습니다. Claude가 입력을 기다리거나 작업이 완료되면 푸시 알림으로 알려줍니다. 이 장에서는 알림 설정 방법과 알림을 활용해 효율적으로 작업을 관리하는 방법을 배웁니다.

<hr>

## 푸시 알림이 오는 시점

Claude Code는 다음 상황에서 모바일 알림을 전송합니다.

| 알림 유형 | 발생 조건 |
|-----------|-----------|
| 입력 대기 | Claude가 다음 지시를 기다릴 때 |
| 도구 승인 필요 | 실행 권한 승인이 필요할 때 |
| 작업 완료 | 긴 작업이 완전히 끝났을 때 |
| 오류 발생 | Claude가 오류를 만나 중단되었을 때 |
| 네트워크 재연결 | 오프라인 후 세션이 복구되었을 때 |

이 알림 덕분에 긴 빌드나 테스트 실행, 파일 분석 같은 작업을 맡기고 다른 일을 하다가 완료 시점에만 대응할 수 있습니다.

<hr>

## 알림 권한 설정

### iOS

**처음 앱 실행 시**

앱을 처음 설치하고 Code 기능에 접근하면, 알림 권한 요청 팝업이 자동으로 나타납니다. **허용** 을 탭합니다.

![iOS 알림 권한 요청 팝업 - "허용" 버튼 강조](../assets/05-5-ios-permission.png)

**나중에 권한을 켜는 방법**

권한 요청을 거부했거나 나중에 활성화하려면:

1. iPhone **설정** 앱 열기
2. 아래로 스크롤하여 **Claude** 탭
3. **알림** → **알림 허용** 토글 켜기
4. **알림 스타일** 에서 **배너** 또는 **알림** 선택 권장

![iPhone 설정 > Claude > 알림 화면](../assets/05-5-ios-notif-settings.png)

### Android

**처음 앱 실행 시**

Android 13 이상에서는 앱 첫 실행 시 알림 권한 요청 팝업이 나타납니다. **허용** 을 탭합니다.

![Android 알림 권한 요청 팝업](../assets/05-5-android-permission.png)

**나중에 권한을 켜는 방법**

1. 기기 **설정** → **앱** (또는 **애플리케이션**)
2. **Claude** 선택
3. **알림** → **알림 허용** 토글 켜기

![Android 설정 > Claude > 알림 화면](../assets/05-5-android-notif-settings.png)

<hr>

## 앱 내 알림 설정

Claude 앱 자체에서도 알림 종류를 세밀하게 조정할 수 있습니다.

**설정 경로:**
- 앱 우측 상단 프로필 아이콘 → **Settings** → **Notifications**

![Claude 앱 Notifications 설정 화면](../assets/05-5-app-notif-settings.png)

| 설정 항목 | 설명 |
|-----------|------|
| Input required | Claude가 입력을 기다릴 때 알림 |
| Tool approval | 도구 승인 필요 시 알림 |
| Task complete | 작업 완료 시 알림 |
| Error alerts | 오류 발생 시 알림 |
| Sound | 알림음 켜기/끄기 |
| Vibration | 진동 켜기/끄기 |

> **권장 설정:** 장시간 작업 중에는 **Input required** 와 **Tool approval** 만 켜두면 불필요한 알림을 줄일 수 있습니다.

## 알림 받고 신속하게 대응하기

### 잠금 화면에서 바로 처리

알림이 오면 잠금 화면에 미리보기가 표시됩니다.

```
[Claude] dev-machine-elegant-fox
도구 실행 승인이 필요합니다.
Bash: npm run test
```

![잠금 화면의 Claude 알림 카드](../assets/05-5-lockscreen.png)

**iOS:** 알림을 왼쪽으로 스와이프하면 **보기** 버튼이 나타납니다. 탭하면 앱이 열리며 해당 세션으로 바로 이동합니다.

**Android:** 알림을 탭하면 해당 세션이 바로 열립니다. 알림을 길게 누르면 **승인 / 거부** 액션 버튼이 나타납니다(지원 기기에 따라 다름).

![Android 알림 확장 - 액션 버튼 표시](../assets/05-5-android-expand.png)

<hr>

### 알림 센터에서 한번에 확인

여러 세션의 알림이 쌓였을 때 알림 센터에서 한눈에 확인합니다.

**iOS:** 화면 상단에서 아래로 스와이프 → 알림 센터

**Android:** 화면 상단에서 아래로 스와이프 → 알림 패널

![알림 센터에 여러 Claude 세션 알림이 쌓인 모습](../assets/05-5-multiple-notifs.png)

## 실전 활용 시나리오

### 시나리오 1 — 빌드 완료 확인

```
1. 모바일에서 Claude에게 지시:
   "프론트엔드 프로젝트 빌드하고 에러 있으면 수정해줘"

2. 스마트폰을 내려놓고 다른 작업

3. 10분 후 알림 수신:
   "dev-machine: 빌드 성공, 입력 대기 중"

4. 알림 탭 → 결과 확인 → 다음 지시
```

### 시나리오 2 — 이동 중 승인

```
1. 출근 중 전철 안에서 알림 수신:
   "도구 승인 필요 — git push origin main"

2. 알림 탭 → 변경 사항 확인

3. 내용 검토 후 승인 또는 거부

4. Claude가 작업 재개 → 완료 시 재알림
```

![이동 중 스마트폰으로 승인하는 화면](../assets/05-5-mobile-approve.png)

## 방해 금지 모드와 알림

야간이나 집중 작업 시간에는 방해 금지 모드를 활용합니다.

**iOS 집중 모드 설정:**
1. **설정** → **집중 모드** → **방해 금지**
2. **앱 알림 허용** 에서 Claude 추가
3. 설정 시간 동안 Claude 알림만 허용

**Android 방해 금지 설정:**
1. **설정** → **소리** → **방해 금지**
2. **예외** 에서 Claude 알림을 **허용된 앱** 으로 추가

이렇게 하면 야간에도 도구 승인이 필요한 경우에만 알림을 받을 수 있습니다.

<hr>

## 알림이 오지 않을 때 확인사항

**체크리스트:**

- [ ] iOS/Android 시스템 알림 권한이 켜져 있는가
- [ ] Claude 앱 내 알림 설정에서 원하는 항목이 활성화되어 있는가
- [ ] 방해 금지 모드가 Claude를 차단하고 있지 않은가
- [ ] 모바일이 세션에 연결된 상태인가 (앱이 백그라운드에 있어도 알림은 수신됨)
- [ ] PC 터미널이 열린 채 Remote Control이 실행 중인가

## 5장 마무리

이 장에서는 스마트폰으로 Claude Code를 완전히 제어하는 방법을 단계별로 배웠습니다.

| 장 | 핵심 내용 |
|----|-----------|
| 5-1 | Claude 모바일 앱 설치 및 계정 연결 |
| 5-2 | QR 코드로 1초 만에 세션 연결 |
| 5-3 | 세션 목록에서 팀 에이전트 선택 |
| 5-4 | 지시 입력 및 도구 승인 처리 |
| 5-5 | 푸시 알림으로 비동기 작업 관리 |

Remote Control과 모바일 앱을 함께 활용하면, PC 앞에 있지 않아도 Claude Code의 모든 기능을 온전히 사용할 수 있습니다. 긴 작업을 맡기고 외출하거나, 이동 중에 승인 처리를 하거나, 팀 에이전트 여러 명을 동시에 관리하는 것이 모두 스마트폰 하나로 가능해집니다.
