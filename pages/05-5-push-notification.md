## 05-5. 푸시 알림으로 작업 완료 확인

## 이 장에서 배울 내용

Claude에게 오래 걸리는 작업을 맡겼을 때, 스마트폰을 계속 들여다볼 필요가 없습니다. Claude가 입력을 기다리거나 작업이 완료되면 푸시 알림으로 알려줍니다. 이 장에서는 알림 설정 방법과 알림을 활용해 효율적으로 작업을 관리하는 방법을 배웁니다.

> 이 장을 마치면 Claude에게 작업을 맡기고 스마트폰을 내려놓은 뒤, 알림이 울릴 때만 돌아와 대응하는 비동기 워크플로우를 실전에서 활용할 수 있게 됩니다.

> 💡 **알림이 사람을 부르는 구조입니다.** 긴 작업을 Claude에게 맡긴 뒤 화면을 계속 들여다볼 필요가 없습니다. 작업 완료·오류·승인 요청 등 판단이 필요해지는 순간에 스마트폰 푸시 알림이 대신 알려줍니다. 일을 맡기고 다른 것을 하다가 알림이 울릴 때만 돌아오면 됩니다.

<hr>

## 푸시 알림이 오는 시점

Claude Code는 다음 상황에서 모바일 알림을 전송합니다.

| 알림 유형 | 발생 조건 |
|-----------|-----------|
| 입력 대기 | Claude가 다음 지시를 기다릴 때 |
| 도구 승인 필요 | 실행 권한 승인이 필요할 때 |
| 작업 완료 | 긴 작업이 완전히 끝났을 때 |
| 오류 발생 | Claude가 오류를 만나 중단되었을 때 |
| 네트워크 재연결 | 오프라인 후 세션이 복구되었을 때 |

이 알림 덕분에 긴 빌드나 테스트 실행, 파일 분석 같은 작업을 맡기고 다른 일을 하다가 완료 시점에만 대응할 수 있습니다.

> 💡 비동기 작업이란 결과가 나올 때까지 화면 앞에서 기다리지 않고, 일을 맡겨 둔 채 다른 활동을 하다가 알림이 오면 그때 대응하는 방식입니다. 푸시 알림이 바로 이 비동기 작업을 가능하게 해 줍니다.

> 💡 택배를 보낸 뒤 앱을 계속 들여다보지 않아도 "배송 완료" 알림이 오면 그때 확인하죠. Claude 푸시 알림도 같습니다. 작업을 맡겨 두고 알림이 울리면 그때 돌아오면 됩니다.

지시를 보내고 휴대폰을 내려놓았다가, 푸시가 오면 탭해서 대응하고 다시 다음 지시로 돌아가면 됩니다. 화면을 지키고 앉아 있는 대신 알림이 나를 부르는 셈입니다. 위 표의 다섯 경우는 모두 사람의 판단이나 확인이 필요해진 순간이라, 알림만 따라가도 일이 막힌 지점을 놓치지 않습니다.

**알림 유형별 대응 방법:**

| 알림 유형 | 권장 대응 |
|-----------|-----------|
| 입력 대기 | 다음 지시를 입력하거나, 계속 진행하라고 응답 |
| 도구 승인 필요 | 실행 내용을 확인 후 승인 또는 거부 |
| 작업 완료 | 결과를 확인하고 다음 단계 지시 |
| 오류 발생 | 오류 메시지를 읽고 해결 방향을 지시 |
| 네트워크 재연결 | 세션 상태를 확인하고 필요시 재지시 |

## 알림 권한 설정

### iOS

**처음 앱 실행 시**

앱을 처음 설치하고 Code 기능에 접근하면, 알림 권한 요청 팝업이 자동으로 나타납니다. **허용** 을 탭합니다.

> 💡 **처음 팝업을 거부했다면?** iOS에서는 한 번 거부하면 앱이 자동으로 다시 묻지 않습니다. 아래 "나중에 권한을 켜는 방법"을 따라 수동으로 활성화해야 합니다.

**나중에 권한을 켜는 방법**

권한 요청을 거부했거나 나중에 활성화하려면:

1. iPhone **설정** 앱 열기
2. 아래로 스크롤하여 **Claude** 탭
3. **알림** → **알림 허용** 토글 켜기
4. **알림 스타일** 에서 **배너** 또는 **알림** 선택 권장

> **알림 스타일 차이:** **배너**는 화면 상단에 잠깐 나타났다 사라지고, **알림**은 탭하기 전까지 잠금 화면에 유지됩니다. 자주 놓치는 편이라면 **알림** 스타일을 권장합니다.

**소리와 배지 설정 (선택):**
- **소리:** 알림음이 울립니다. 회의 중이나 조용한 환경에서는 끄고 진동만 켜두세요.
- **배지:** 앱 아이콘에 숫자 뱃지가 붙습니다. 대기 중인 승인 건수를 한눈에 확인할 수 있습니다.

### Android

**처음 앱 실행 시**

Android 13 이상에서는 앱 첫 실행 시 알림 권한 요청 팝업이 나타납니다. **허용** 을 탭합니다.

**나중에 권한을 켜는 방법**

1. 기기 **설정** → **앱** (또는 **애플리케이션**)
2. **Claude** 선택
3. **알림** → **알림 허용** 토글 켜기

> 💡 **Android 제조사별 차이.** 삼성 One UI, LG UX 등 제조사 커스텀 UI에서는 설정 메뉴 이름이 다를 수 있습니다. "앱" 또는 "애플리케이션 관리"에서 Claude를 찾으면 됩니다. 알림이 여전히 오지 않는다면 "배터리 최적화" 설정에서 Claude를 예외로 추가하세요 — 일부 기기는 배터리 절약을 위해 백그라운드 알림을 차단합니다.

## 앱 내 알림 설정

Claude 앱 자체에서도 알림 종류를 세밀하게 조정할 수 있습니다.

**설정 경로:**
- 앱 우측 상단 프로필 아이콘 → **Settings** → **Notifications**

| 설정 항목 | 설명 |
|-----------|------|
| Input required | Claude가 입력을 기다릴 때 알림 |
| Tool approval | 도구 승인 필요 시 알림 |
| Task complete | 작업 완료 시 알림 |
| Error alerts | 오류 발생 시 알림 |
| Sound | 알림음 켜기/끄기 |
| Vibration | 진동 켜기/끄기 |

> **권장 설정:** 장시간 작업 중에는 **Input required** 와 **Tool approval** 만 켜두면 불필요한 알림을 줄일 수 있습니다.

**상황별 추천 설정:**

| 상황 | 켜 두면 좋은 항목 |
|------|------------------|
| 빌드·테스트 자동화 중 | Task complete, Error alerts |
| 긴 파일 분석 맡길 때 | Input required, Task complete |
| 민감한 작업(파일 삭제 등) | Tool approval |
| 야간 방치 작업 | Error alerts만 (방해 금지 모드 연동) |

> 💡 알림이 너무 자주 오면 결국 전부 무시하게 됩니다. 정말 대응이 필요한 순간에만 울리도록 항목을 줄여 두는 편이 장기적으로 더 효율적입니다.

## 알림 받고 신속하게 대응하기

### 잠금 화면에서 바로 처리

알림이 오면 잠금 화면에 미리보기가 표시됩니다.

```
[Claude] dev-machine-elegant-fox
도구 실행 승인이 필요합니다.
Bash: npm run test
```

잠금 화면에서도 위 미리보기처럼 어느 세션이(`dev-machine-elegant-fox`) 무엇을(`Bash: npm run test`) 승인받으려는지 바로 보입니다. 굳이 앱을 열지 않아도 알림을 길게 누르면 그 자리에서 **승인·거부** 버튼이 펼쳐져, 잠금 화면 위에서 곧장 결정할 수 있습니다. 더 자세히 보고 싶을 때만 아래 방법으로 앱을 열어 해당 세션으로 들어가면 됩니다.

**iOS:** 알림을 왼쪽으로 스와이프하면 **보기** 버튼이 나타납니다. 탭하면 앱이 열리며 해당 세션으로 바로 이동합니다.

**Android:** 알림을 탭하면 해당 세션이 바로 열립니다. 알림을 길게 누르면 **승인 / 거부** 액션 버튼이 나타납니다(지원 기기에 따라 다름).

> 💡 **미리보기 텍스트가 보이지 않는다면?** iOS에서는 설정 → Claude → 알림 → "알림 미리보기 표시"를 **항상** 또는 **잠금 해제 시**로 설정해야 미리보기가 잠금 화면에 표시됩니다. 기본값이 "잠금 해제 시"이므로 Face ID 또는 지문 인식 후에만 내용이 보일 수 있습니다.

<hr>

### 알림 센터에서 한번에 확인

여러 세션의 알림이 쌓였을 때 알림 센터에서 한눈에 확인합니다.

**iOS:** 화면 상단에서 아래로 스와이프 → 알림 센터

**Android:** 화면 상단에서 아래로 스와이프 → 알림 패널

알림 센터에서는 세션별로 묶인 알림들을 한 번에 볼 수 있습니다. 여러 Claude 세션을 동시에 운영 중이라면 어느 세션이 대응을 기다리는지 여기서 빠르게 파악하세요.

> 💡 **여러 세션 동시 관리 팁.** 세션 이름을 `project-alpha`, `project-beta`처럼 구별하기 쉽게 지정해 두면, 알림 센터에서 어느 프로젝트의 알림인지 즉시 알 수 있습니다. 세션 이름 지정 방법은 5-3장을 참고하세요.

## 실전 활용 시나리오

### 시나리오 1 — 빌드 완료 확인

```
1. 모바일에서 Claude에게 지시:
   "프론트엔드 프로젝트 빌드하고 에러 있으면 수정해줘"

2. 스마트폰을 내려놓고 다른 작업

3. 10분 후 알림 수신:
   "dev-machine: 빌드 성공, 입력 대기 중"

4. 알림 탭 → 결과 확인 → 다음 지시
```

빌드가 도는 2~15분 동안 다른 작업에 집중할 수 있습니다. Claude가 오류를 발견해 수정까지 마쳤다면 알림 한 번으로 전 과정이 처리됩니다.

### 시나리오 2 — 이동 중 승인

```
1. 출근 중 전철 안에서 알림 수신:
   "도구 승인 필요 — git push origin main"

2. 알림 탭 → 변경 사항 확인

3. 내용 검토 후 승인 또는 거부

4. Claude가 작업 재개 → 완료 시 재알림
```

이동 중에도 개발 흐름이 끊기지 않습니다. 한 손으로 알림을 처리하고 다시 내려놓으면, Claude는 PC에서 계속 작업을 이어갑니다.

### 시나리오 3 — 야간 자동화 후 아침 확인

```
1. 퇴근 전 지시:
   "전체 테스트 돌리고 실패한 케이스 분석해서 리포트 작성해줘"

2. 스마트폰을 잠그고 귀가

3. 자는 동안 Claude가 작업 완료

4. 아침 기상 시 알림 확인:
   "작업 완료 — test-report.md 생성됨"

5. 출근길에 리포트 내용 확인 → 수정 지시
```

자는 동안 테스트가 돌아 아침엔 결과만 확인하면 됩니다. 하루를 곧장 시작할 준비가 되어 있는 셈입니다.

## 방해 금지 모드와 알림

야간이나 집중 작업 시간에는 방해 금지 모드를 활용합니다.

**iOS 집중 모드 설정:**
1. **설정** → **집중 모드** → **방해 금지**
2. **앱 알림 허용** 에서 Claude 추가
3. 설정 시간 동안 Claude 알림만 허용

> 💡 **집중 모드 예약.** iOS 집중 모드는 특정 시간대(예: 오후 11시~오전 7시)에 자동으로 켜지도록 예약할 수 있습니다. 이렇게 하면 야간에는 오류 알림만 받고, 낮에는 모든 알림을 수신하는 방식으로 자동화됩니다.

**Android 방해 금지 설정:**
1. **설정** → **소리** → **방해 금지**
2. **예외** 에서 Claude 알림을 **허용된 앱** 으로 추가

이렇게 하면 야간에도 도구 승인이 필요한 경우에만 알림을 받을 수 있습니다.

**방해 금지 모드와 Claude 알림 조합 전략:**

| 시간대 | 권장 설정 |
|--------|-----------|
| 업무 시간 | 모든 알림 허용 |
| 집중 작업 중 | Tool approval·Error alerts만 예외 |
| 야간 방치 자동화 | Error alerts만 예외, 나머지 차단 |
| 회의 중 | 방해 금지 ON, Claude 예외 없음 |

## 알림이 오지 않을 때 확인사항

**체크리스트:**

- [ ] iOS/Android 시스템 알림 권한이 켜져 있는가
- [ ] Claude 앱 내 알림 설정에서 원하는 항목이 활성화되어 있는가
- [ ] 방해 금지 모드가 Claude를 차단하고 있지 않은가
- [ ] 모바일이 세션에 연결된 상태인가 (앱이 백그라운드에 있어도 알림은 수신됨)
- [ ] PC 터미널이 열린 채 Remote Control이 실행 중인가

**추가 확인:**
- [ ] Android의 경우 배터리 최적화에서 Claude가 예외로 설정되어 있는가
- [ ] iOS의 경우 "알림 미리보기 표시" 설정이 원하는 대로 되어 있는가
- [ ] 여러 기기에 Claude가 설치된 경우, 알림을 받을 기기가 올바르게 선택되어 있는가

> 💡 **테스트 알림 보내기.** 설정이 올바른지 확인하려면 Claude에게 짧게 "5초 후에 완료 알림 보내줘" 같은 지시를 내린 뒤 화면을 잠그고 기다려보세요. 알림이 오면 설정이 정상입니다.

## 05장 마무리

이 장에서는 스마트폰으로 Claude Code를 완전히 제어하는 방법을 단계별로 배웠습니다.

| 장 | 핵심 내용 |
|----|-----------|
| 5-1 | Claude 모바일 앱 설치 및 계정 연결 |
| 5-2 | QR 코드로 1초 만에 세션 연결 |
| 5-3 | 세션 목록에서 팀 에이전트 선택 |
| 5-4 | 지시 입력 및 도구 승인 처리 |
| 5-5 | 푸시 알림으로 비동기 작업 관리 |

Remote Control과 모바일 앱을 함께 활용하면, PC 앞에 있지 않아도 Claude Code의 모든 기능을 온전히 사용할 수 있습니다. 긴 작업을 맡기고 외출하거나, 이동 중에 승인 처리를 하거나, 팀 에이전트 여러 명을 동시에 관리하는 것이 모두 스마트폰 하나로 가능해집니다.
